## Why this exists

The dominant failure mode in agent-memory research is **building a benchmark the feature cannot lose**. LongMemEval/LoCoMo are verbatim-answerable with engineered clusters — any summarizer "wins." MemoryBench (arXiv:2510.17281) found no advanced memory system reliably beats plain all-context RAG; CTIM-Rover (arXiv:2505.23422) found distilled insights *degraded* agents by ~11 points. The honest prior is that this feature **fails**. The evaluation is designed so that rejection is the expected, successful outcome.

KAOS evaluating KAOS is circular unless three locks hold. They are encoded in the ISA.lock and summarized here.

## The three anti-self-deception locks

1. **Grader-blind.** Per-query correctness is judged by `kaos.metaharness.verifier.SurrogateVerifier`, which by construction sees task + outputs but not the harness/feature source — and is fed an arm-anonymized, shuffled result stream so it cannot know whether synthesis was on. (KAOS already ships this isolation property for exactly this reason.)
2. **Separate agents.** The agent that generates the episodic workload and the agent that generates the held-out queries run in distinct KAOS sessions, import neither the synthesis module nor each other. The complex problem is not co-designed with the solution.
3. **Gate-first.** The harness must be proven able to FAIL before any synthesis code is written: run `B0..B3` with `FULL` stubbed to equal `B0`; the harness MUST emit `[KILL: G1]`. A harness in which the feature cannot lose is broken, and its later "pass" is inadmissible.

A run is additionally **VOID** (neither pass nor fail) if the LLM-judge disagrees with a frozen 50-query human-labeled sample below κ=0.85, or if `G0` (the all-arms verbatim-recall sanity floor) fails.

## The complex problem (not a toy)

A simulated multi-month, multi-agent incident-response operation: ~40 KAOS agents over a simulated quarter against a fake distributed system. Episodic memory is written **organically** by agents doing the work; the Hebbian `associations` graph forms from genuine co-retrieval. **Clusters are discovered, never hand-placed** — a workload that pre-places cluster boundaries VOIDs the run. Includes ≥20% distractor clusters (plausible, wrong-if-summarized-naively) and a temporal-contradiction stream (later incidents invalidate earlier root causes).

Four held-out query classes (≥200 each):

| Class | Trivial baselines can answer? | Purpose |
|---|---|---|
| verbatim_recall | yes | sanity floor (G0) |
| compositional_multihop | no — answer lexically absent from every episode *and* every single cluster | the real test |
| abstraction_only | no — cue/trigger semantically disconnected | the real test |
| tail_fact_probe | yes (one rare detail) | anti-washout guard (G5) |

## Arms (each is a real KAOS lever)

`B0` no-synthesis · `B1` non-LLM Jaccard dedup · `B2` keep-everything (position-randomized) · `B3` cost-matched (same LLM budget on query expansion, not consolidation) · `L1` synthesize-but-never-retrieve · `L2` random-insight · `L3` answer-masked synthesizer · `FULL`. One seeded DB forked 8 ways; the only variable is the memory layer.

## The kill gates (summary — ISA.lock is authoritative)

- **G0** sanity floor — every arm ≥0.95 on verbatim_recall, else **VOID**.
- **G1** beats cheap baselines — FULL − max(B0,B1,B3) ≥ **+0.10** on compositional+abstraction, bootstrap 95% CI excludes 0, n≥200/class. *Kill.*
- **G2** stored artifact is causal — synthesized insights in top-K for ≥**60%** of queries they help, AND lesion L1 reproduces **≤20%** of the gain. *Kill.*
- **G3** not merely the free call — FULL − B3 ≥ **+0.07**. *Kill.*
- **G4** survives distribution shift — ≥**70%** of in-distribution gain retained on the held-out shift split, and FULL−B0 strictly >0 on every class there. *Kill.*
- **G5** no leakage / no washout — answer-masked L3 retains ≥**80%** of gain, tail-fact probe regresses **≤0.03** vs B0. *Kill.*

**Verdict rule:** VOID if G0 or judge-audit fails; otherwise ACCEPT only if G1∧G2∧G3∧G4∧G5. Any single kill-gate failure REJECTS outright. No retune-and-rerun.

## Build protocol (the order is the point)

1. **Harness + ISA.lock committed first** — done at this hash, no feature code present.
2. **Workload agent + query agent** (separate sessions, synthesis-blind). Fill the two `TO BE FILLED` manifests (`judge_audit_sample`, `shift_split_manifest`) → supersede with `ISA.lock.v2.json` (new hash, new pre-registration commit) **before any FULL run**.
3. **Gate-first falsification**: run B0–B3 with FULL≡B0; assert the harness emits `[KILL: G1]`. Commit that proof.
4. **Judge audit**: 50-query human labels vs SurrogateVerifier; require κ≥0.85 or fix the judge.
5. **Only now** implement `kaos/dream/phases/synthesis.py` (deterministic clustering core, opt-in `llm_call_fn` fallback, fingerprint-cached `synth:` prefix in `llm_diagnosis_cache`, writes `insight` memory + associations links).
6. Run all 8 arms on the frozen seed. The harness exit code is the binding verdict.

## Honest expected outcomes

- **~50%** G1 or G3 fails → **reject**. Successful evaluation.
- **~30%** G1/G3 pass but G2 fails → gain is the synthesis-time reasoning, not the artifact → reframe as "a reasoning pass during dream," do **not** ship as synthesis-as-consolidation.
- **~20%** all gates pass → ship with numbers, and the writeup still states plainly: this is *better lookup, not weight-based generalization*; arXiv:2604.27707's ceiling is untouched (the v0.8.3 honesty standard).

## Effort

Harness + locks + workload/query agents + gate-first falsification proof + judge audit: **~1.5–2 weeks**, deliberately more than the feature itself. The feature (`synthesis.py`) is only written at step 5, and only if step 3 proved the harness can kill it.

## Out of scope

Weight-based / parametric consolidation (the paper's actual prescription — KAOS does not do parameter updates). Compositional generalization beyond retrieval. SAGE-style trained GFM/RL writer (rejected, arXiv:2605.12061 evaluation: 2/10 for KAOS — GPU training, embeddings, and weakest exactly on long-term agent memory).
