# KAOS v0.9 — Release Plan

**Status:** planned (not yet implemented)
**Type:** maintenance + queryability + a meta-deliverable, plus ONE pre-registered probe
**One-line:** fix the P0 that broke the meta-harness for CLI proposers, make experiments queryable, codify the falsifiable-eval apparatus as a first-class KAOS primitive, and run one honest pre-registered probe whose verdict will be reported either way.

---

## Why this release is deliberately small

v0.8.3 over-delivered (quality score + failure taxonomy + critical-step localizer + ISA/ISC + war-room UI in one cycle). The repo has spent its mechanism-debt budget. Meanwhile a real P0 user-facing bug (#11 — Claude CLI proposer timeouts) is unfixed, AI-reported #8 (experiment tracking) is open, and **five evaluated mechanism ideas have been rejected or parked this cycle** (SAGE, the synthesis-as-consolidation arc, AutoResearchClaw, HASP, Life-Harness mechanism). Shipping mechanism #N+1 on top of an unfixed P0 is theater.

The session-proven discipline (4 REJECT + 1 PARK out of 5 evaluated, by pre-registered falsifiable evaluation) IS the differentiator. v0.9 productizes that discipline rather than adding more mechanism. The release narrative is **reliability, queryability, and a falsifiable-eval primitive**.

## The 4 PRs (merge order)

### PR-1 — `proposer-streaming` (fixes P0 #11)

**Closes** [#11](https://github.com/canivel/kaos/issues/11) — Claude CLI proposer times out every iteration, blocking the meta-harness in practice.

**Do not** migrate the `claude_code` provider path to the direct Anthropic API. The existing CLI/SDK split is a deliberate architectural choice; replacing it papers over the real issue (the subprocess call is blocking without progress signal).

**Do**:

- Switch the `claude_code` provider in `kaos/router/providers.py` to the CLI's existing streaming mode (`--output-format stream-json` or equivalent). Consume incrementally.
- Add two timeout dimensions:
  - **Idle timeout** (no new tokens for N seconds) → raises `ProposerStalled`, recoverable.
  - **Wall timeout** (absolute upper bound) → hard kill.
- Meta-harness search loop (`kaos/metaharness/search.py`) catches `ProposerStalled`, logs the partial output to `dream_runs`, marks the iteration `incomplete`, continues to next iteration instead of dying. (Empty/partial responses remain *errors* per the existing coding rule — but the *loop* survives.)
- New CLI: `kaos doctor proposer` — runs a 30-second smoke prompt against every configured provider, reports latency + stream health. Cheap regression catcher.

**Tests** (`tests/test_proposer_streaming.py`):
- Fake subprocess emitting chunks slowly → runner accumulates without hanging past idle bound.
- Wall timeout fires when stream stalls beyond limit.
- `ProposerStalled` is caught by the search loop; the loop advances to the next iteration.
- `kaos doctor proposer` reports a stalled provider rather than hanging.

**No schema change.** Effort: 3–5 days.

### PR-2 — `eval-harness-module` (codify the falsifiable apparatus)

The apparatus that **rejected 4 of 5 mechanism ideas this cycle** — `demo_synthesis_consolidation_bench/{gates,judge,falsify,workload,queries,arms}.py` — gets refactored into a first-class KAOS primitive.

**New module** `kaos/eval/harness/` (same submodule names: `gates.py`, `judge.py`, `falsify.py`, `probe.py`):

- `kaos.eval.harness.Probe` — base class. Subclasses define `arms()`, `workload()`, `gates()`, `falsify()`.
- The runner **enforces hash-locking the manifest before execution**, **blocks goalpost moves** (manifest hash mismatch = abort), writes `results.json` in a canonical schema, and **refuses to overwrite a results file whose manifest hash matches** (prevents silent re-runs).
- `kaos.eval.harness.judge.SurrogateVerifier` — the blind heuristic judge primitive, already proven through synthesis arc.

**New CLI**:
- `kaos eval probe run <probe.py>` — runs a probe, writes hash-locked `results.json`.
- `kaos eval probe verify <results.json>` — confirms the result file matches its declared manifest hash.
- `kaos eval probe falsify <probe.py>` — runs the falsification self-test (the gate-first kill proof).

**Migration of the existing demo**: `demo_synthesis_consolidation_bench/` stays as-is (rejected-research artifact, audit trail). PR-2 demonstrates the new module by **porting that bench to use `kaos.eval.harness`** as the canonical first consumer (re-running it should produce byte-identical hash-locked manifests; if it doesn't, the refactor is broken).

**Tests**: harness self-test (the manifest-hash-mismatch abort path), probe lifecycle (init → falsify → run → verify), goalpost-block detection.

**No schema change.** Effort: 3–4 days.

**Meta-deliverable framing**: KAOS becomes the first agent framework that ships its own kill-switch primitive. The thing that has been rejecting features for us is now a CLI command anyone can use.

### PR-3 — `experiment-cli` (closes #8)

**Closes** [#8](https://github.com/canivel/kaos/issues/8) — Experiment tracking layer.

**Skeptical scope**: this is a **navigation layer over existing artifacts**, NOT a new experiment platform. KAOS already has `dream_runs`, the `results.json` convention, MemoryStore, and meta-harness search results. A new `experiments` table that *duplicates* metric data would drift and rot.

**Schema v8 → v9** (additive, one table):
```sql
CREATE TABLE IF NOT EXISTS experiments (
    experiment_id INTEGER PRIMARY KEY AUTOINCREMENT,
    name          TEXT UNIQUE NOT NULL,
    ref_kind      TEXT NOT NULL CHECK (ref_kind IN ('dream_run','results_json','search_run')),
    ref_value     TEXT NOT NULL,          -- dream_runs.run_id, or filesystem path, etc.
    manifest_hash TEXT,                    -- sha256 of the eval manifest if applicable
    tags          TEXT NOT NULL DEFAULT '[]',
    created_at    TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%f','now'))
);
CREATE INDEX IF NOT EXISTS idx_experiments_tags ON experiments(tags);
```

**CLI**:
- `kaos experiment log <name> --results <path> [--tags x,y] [--manifest-hash ...]` — writes a pointer row. **No metric duplication.**
- `kaos experiment list [--tag x] [--since YYYY-MM-DD]` — index view.
- `kaos experiment compare <name1> <name2> [--metric M]` — reads the referenced `results.json` files, diffs the metric block, prints a delta table. Pure read-side; no recomputation.
- `kaos experiment show <name>` — prints manifest hash, gates passed/failed (if PR-2-style), verdict line from the originating commit message if present.

**MCP**: 0 new tools (50 is enough; the CLI surface is sufficient).

**Tests**: round-trip log/list/show, compare on two real bench results files, manifest-hash-mismatch detection.

**Effort: 2–3 days**.

### PR-4 — `action-realization-probe` (Life-Harness narrow slice, PROBE ONLY)

**This is the only mechanism-adjacent work in v0.9.** Per session discipline, **no mechanism code ships in v0.9** — only the pre-registered probe whose verdict will determine v0.10.

**The candidate (from Life-Harness arXiv:2605.22166)**: a pre-execution Action Realization Layer wrapping the tool registry — deterministic validation/canonicalization/blocking of malformed tool calls before they execute. KAOS's v0.8.3 critical-step localizer is *post-mortem* (after failure); this would be *pre-failure*. That timing difference is the one place the v0.8.3 native baseline cannot trivially absorb.

**Pre-registered, hash-locked under `demo_action_realization_bench/ISA.lock.json`** (committed BEFORE any mechanism code exists; same discipline as the synthesis arc):

- **Workload**: 200 organic failure incidents from existing `dream_runs` where the v0.8.3 localizer classified failure as `action`-class (malformed tool call, schema violation, unparseable args). Real distribution from real KAOS use, **not synthetic**.
- **Baseline arm (B1)**: v0.8.3 native — localizer + diagnoser + retry-with-feedback. **This is the real bar.**
- **Treatment arm (FULL)**: Action Realization Layer — pre-execution validation/canonicalization that blocks malformed calls before the tool boundary.
- **Lesion arms**:
  - **L1** never-fire (registered but `should_activate=False`) — isolates "library presence" from "intervention".
  - **L2** random-fire — controls for "any intervention helps".
- **Control slice**: 200 non-action failures (localizer classified `planning` or `memory`). Treatment must NOT regress these.

**Pre-committed kill gates** (frozen in `ISA.lock.json`, hash-locked, NO retune):
- **G1** treatment > B1 by **≥ +4.0pp** completion rate on the action-class slice, p<0.05 (paired bootstrap, n≥200). The real bar — must beat KAOS-v0.8.3-native, not naive.
- **G2** treatment > L1 (never-fire) by **≥ +3.0pp** AND > L2 (random-fire) by **≥ +2.0pp**. Causal isolation.
- **G3** treatment ≥ B1 − 1.0pp on the non-action control slice. **No regression where it shouldn't help** (the SWE-Skills-Bench failure mode where 3/49 deployed skills *degraded* performance).
- **G4** inline overhead p95 < 500 µs per tool call (KAOS's two-timescale rule).

**Verdict rule**: ALL FOUR must pass → **PASS** opens a v0.10 milestone for the mechanism. Any single failure → **REJECT** and logged alongside the other 5 evaluated rejections. **No retune-and-rerun.**

**Reuses PR-2's `kaos.eval.harness` module** — second consumer, demonstrates the primitive isn't single-use.

**Effort: 3–4 days** (probe only; mechanism deferred to v0.10 conditional on G1∧G2∧G3∧G4).

**Honest expected outcome**: REJECT is likely (~0.55 confidence) per the SWE-Skills-Bench negative prior and the strong KAOS-v0.8.3-native baseline. **REJECT is a successful outcome.** A PASS would be the first time anything in this entire evaluation arc beat the bar — and would earn v0.10.

## Risks

- **Scope creep** (largest risk, mitigated by this frozen spec).
- **PR-1 streaming** depends on the Claude CLI's streaming mode being reliable across versions; fallback is the wall-timeout-with-loop-survival path which is already in scope.
- **PR-2 API design** is the only PR where the public surface matters long-term; a bad `Probe` interface ages badly. Mitigation: port the existing bench as the *first* consumer before tag, prove the API actually works on a real probe.
- **PR-4 may VOID** (e.g., the action-class incident pool is too small or unbalanced). Per discipline, VOID permits harness-plumbing fixes; gates stay byte-frozen. Same recovery pattern as the synthesis arc (which VOIDed twice before rendering a real verdict).
