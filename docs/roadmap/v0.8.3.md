# KAOS v0.8.3 — Release Plan

**Status:** planned (not yet implemented)
**Type:** point release — no major-version ceremony
**One-line:** richer outcome signal, failure understood by reasoning-class with trajectory localization, structured per-task objectives, and (conditionally) cheaper consensus — all under one schema migration.

---

## Why one point release, not three

The three workstreams below were originally scoped as v0.8.3 / v0.9 / v0.10. They are folded into a single v0.8.3 because:

- They share **one additive schema migration** (v7 → v8). Splitting them would mean v8/v9/v10/v11 migrations for changes that are all additive and non-breaking — pointless churn.
- They compose into **one coherent story** ("the framework understands outcomes, failures, and objectives at finer grain"), not three independent features.
- None of them changes a public API in a breaking way. Everything is additive: new columns default to NULL, new functions have optional params, existing call-sites keep working untouched.

The internal track structure (A / B / C) is preserved for PR slicing and sequencing. Track C stays **conditional** — see its gating rule.

---

## Consolidated schema migration (v7 → v8)

All schema changes ship in one `MIGRATION_V8_SQL` block. Additive only; existing rows unaffected.

```sql
-- Track A: quality score
ALTER TABLE skill_uses ADD COLUMN quality REAL
  CHECK (quality IS NULL OR (quality >= 0 AND quality <= 1));
CREATE INDEX IF NOT EXISTS idx_skill_uses_quality
  ON skill_uses(skill_id, quality) WHERE quality IS NOT NULL;

-- Track B1: failure taxonomy
ALTER TABLE failure_fingerprints ADD COLUMN taxonomy_class TEXT
  CHECK (taxonomy_class IS NULL OR taxonomy_class IN
    ('memory','reflection','planning','action','system','unknown'));
ALTER TABLE failure_fingerprints ADD COLUMN taxonomy_subclass TEXT;

-- Track B2: critical-step localizer
CREATE TABLE IF NOT EXISTS critical_steps (
    cs_id           INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_id        TEXT NOT NULL REFERENCES agents(agent_id),
    fingerprint_id  INTEGER REFERENCES failure_fingerprints(fp_id),
    log_position    INTEGER NOT NULL,
    tool_call_id    INTEGER,
    rationale       TEXT,
    method          TEXT,                -- 'heuristic' | 'llm' | 'llm-cached'
    confidence      REAL,
    isc_id          INTEGER,             -- optional link to a failed criterion (Track B3)
    created_at      TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%f','now'))
);
CREATE INDEX IF NOT EXISTS idx_critical_steps_agent ON critical_steps(agent_id);

-- Track B3: ISA / ISC
CREATE TABLE IF NOT EXISTS ideal_states (
    isa_id          INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_id        TEXT NOT NULL REFERENCES agents(agent_id),
    title           TEXT NOT NULL,
    summary         TEXT NOT NULL,
    created_at      TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%f','now')),
    completed_at    TEXT,
    overall_status  TEXT CHECK (overall_status IS NULL OR overall_status IN
                      ('pending','passed','failed','abandoned'))
);
CREATE TABLE IF NOT EXISTS ideal_state_criteria (
    isc_id           INTEGER PRIMARY KEY AUTOINCREMENT,
    isa_id           INTEGER NOT NULL REFERENCES ideal_states(isa_id),
    criterion        TEXT NOT NULL,
    verification     TEXT,
    status           TEXT NOT NULL DEFAULT 'pending'
                     CHECK (status IN ('pending','passed','failed','skipped')),
    failure_taxonomy TEXT,               -- maps to taxonomy_class when status='failed'
    failure_note     TEXT,
    verified_at      TEXT
);
CREATE INDEX IF NOT EXISTS idx_isc_isa ON ideal_state_criteria(isa_id, status);

-- Track C (conditional): Aegean incremental quorum
ALTER TABLE shared_log ADD COLUMN vote_confidence REAL
  CHECK (vote_confidence IS NULL OR (vote_confidence >= 0 AND vote_confidence <= 1));
ALTER TABLE shared_log ADD COLUMN decide_mode TEXT
  CHECK (decide_mode IS NULL OR decide_mode IN ('fixed','incremental'));
```

`schema.py`: bump `SCHEMA_VERSION = 8`, add `MIGRATION_V8_SQL`, add `if from_version < 8: conn.executescript(MIGRATION_V8_SQL)` in `_apply_migrations()`.

If Track C is deferred (gating not met), the two `shared_log` columns still ship in the migration (cheap, additive, forward-compatible) but no code reads them yet.

## Track A — Quality score [0, 1]

**Borrowed from:** PAI (richer outcome signal than binary success).

### Goal
Continuous `quality ∈ [0,1]` on `skill_uses`, fully backward-compatible. Wilson lower bound generalizes to continuous outcomes with no formula change.

### API
`kaos/skills.py`:
```python
def record_outcome(self, skill_id, success, *, agent_id=None,
                    quality: float | None = None, task_hash=None) -> None:
```
- `quality` in `[0,1]` or `ValueError` (no silent clamp).
- `None` → fall back to binary `success`.

`kaos/dream/signals.py` — `weighted_score()` signature unchanged. Internally, effective successes = `SUM(quality)` where non-NULL, else `SUM(success)`. Wilson `(s, n)` accepts real-valued `s`.

### CLI / MCP
- `kaos skills outcome <id> --quality 0.75` (mutually exclusive with `--success`)
- MCP `skill_outcome` gains optional `quality` param

### Tests — `tests/test_quality_score.py` (~8)
binary-only row; quality row; mixed history; out-of-range raises; CLI round-trip; migration leaves old rows NULL; weighted_score prefers quality; Wilson with fractional successes.

### Benchmark — `demo_quality_score_bench/`
Binary baseline vs quality variant on the realistic retrieval workload. Expect similar/higher accuracy with **lower variance** (partial credit denoises the estimator).

### Effort: 1 day

## Track B — Failure understanding + structured objectives

**Borrowed from:** AgentErrorTaxonomy (arXiv:2509.25370) + ISA/ISC pattern (PAI).

### B1 — Taxonomy extension (~3 days)

`failure_fingerprints` gains `taxonomy_class` (memory/reflection/planning/action/system/unknown) + free-form `taxonomy_subclass`. Existing `category` (transient/config/code/infra/unknown) stays — it is execution-flavoured; taxonomy is reasoning-flavoured. Both populated by diagnosers.

`kaos/dream/diagnosis.py`:
- Each of the 8 heuristics assigns `(taxonomy_class, taxonomy_subclass)`.
- `LLMDiagnoser` prompt emits both; cached together in `llm_diagnosis_cache`.
- New `classify_taxonomy(error, context) -> (class, subclass)` for taxonomy-only callers.

CLI: `kaos dream failures --taxonomy-class planning`.
Tests: 12 (8 heuristics + 4 LLM).

### B2 — Critical-step localizer (~5 days)

New phase `kaos/dream/phases/localize.py`. Input: failed `(agent_id, run_id)`. Loads tool_calls + shared_log chronologically. Heuristic-first scoring (distance-from-first-error, write-vs-read, later-contradiction), LLM fallback when confidence < 0.6 (trace-fingerprint cached, same pattern as `LLMDiagnoser`). Writes a `critical_steps` row.

CLI: `kaos dream localize <agent_id>`. MCP: `dream_localize`.
Tests: 10 (`tests/test_critical_step.py`) — early-bad-retrieval / mid-trace-tool-error / late-assertion shapes.
Benchmark: extend `demo_failure_intelligence_bench/scenario.py` with 5 planted-bug-N-steps-before-visible-error trajectories; localizer within ±1 step in ≥4/5.

### B3 — ISA / ISC (~1 week)

New module `kaos/ideal_state.py`:
```python
class IdealStateStore:
    def create(self, agent_id, title, summary, criteria: list[dict]) -> int
    def mark(self, isc_id, status, failure_taxonomy=None, note=None) -> None
    def finalize(self, isa_id) -> str          # all passed→'passed'; any failed→'failed'
    def list_open(self, agent_id=None) -> list[IdealState]
```

**Composition:**
- `mark(failed, failure_taxonomy=...)` uses B1's taxonomy classes → consolidation groups failed ISCs by class to surface systemic patterns.
- A `critical_steps` row may link `isc_id` → trace back from failing criterion to the step it went wrong (B2).
- Per-criterion pass/fail → finer plasticity signal than binary task success, fed via Track A's quality column (an ISA with 4/5 ISCs passed → quality 0.8 on the associated skill use).

CLI: `kaos ideal-state create|mark|show|list`. MCP: `ideal_state_create|mark|get`.
Tests: ~15 (`tests/test_ideal_state.py`).
Benchmark: `demo_isa_isc_bench/` — agents declaring ISAs upfront get higher quality-weighted retrieval accuracy after N sessions.

### Track B effort: ~3 weeks (3 + 5 + 7 days + integration/docs)

## Track C — Aegean incremental quorum (CONDITIONAL)

**Borrowed from:** Aegean (arXiv:2512.20184).

### Gating rule — build the *code* only if at least one holds at implementation time:
- KAOS users run ≥5 concurrent agents in real deployments, **or**
- benchmark/production decide-phase latency >100ms p95, **or**
- an issue explicitly requests faster consensus.

If none hold: the two `shared_log` columns still ship in the v8 migration (forward-compat, zero cost), but `decide()` keeps only `mode='fixed'`. Revisit next release.

### If built
`vote(..., confidence: float | None = None)`; `decide(..., mode='fixed'|'incremental', quorum_threshold=0.95)`.
- Worst-case quorum: stop once trailing side mathematically cannot catch up.
- Confidence-weighted quorum: stop when weighted aggregate exceeds threshold.
`mode='fixed'` is the default and is byte-for-byte the current behaviour (regression-protected).

CLI: `kaos log decide --intent N --mode incremental`. MCP: `shared_log_decide` gains `mode`+`threshold`.
Tests: ~10. Benchmark: `demo_consensus_latency_bench/` — 10-agent synthetic; expect 3–8× decide speedup at N=10, ~0 at N=2, accuracy within 2.5% of fixed.

### Track C effort: 1 week *if* gated in; 0 if deferred (columns ship regardless)

- Track A: 1 day
- Track B: ~3 weeks

## Release narrative

> *v0.8.3 — the framework understands outcomes, failures, and objectives at finer grain. Skill outcomes carry a continuous quality score instead of a coin flip. Failures are classified by reasoning-class and the trajectory step where each one started. Agents declare what "done" looks like upfront, and every criterion becomes a graded signal feeding the same plasticity loop shipped in v0.8.1. No breaking changes, one additive migration.*

## Out of scope (explicitly deferred)

- LLM-generated ISAs from natural-language task descriptions (operators author criteria manually in v0.8.3)
- Cross-agent ISA templates / reuse
- Runtime mid-task ISC enforcement (v0.8.3 verifies only at finalize)
- Byzantine-fault-tolerant or reputation-weighted voting
- Synthesis-as-consolidation (the arXiv:2604.27707 idea) — still parked, separate track
- Typed knowledge graph — still parked (rated 3-4/10), revisit only on explicit demand

## Risks

- **Scope.** Largest point release to date. Mitigation: PR 1 lands the migration alone; tracks are independently revertable; Track C is gated out by default.
- **Wilson on continuous outcomes** is mathematically valid but less standard than Bernoulli — add a sourced note in `signals.py`.
- **Localizer accuracy** is heuristic-first; the benchmark (±1 step in ≥4/5) is the acceptance gate, not vibes.
- **ISA authoring burden** on operators. Mitigation: ISAs are optional; skills/memory/plasticity all work without them. ISA only adds signal when present.
