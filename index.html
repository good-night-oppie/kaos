Neuroplasticity

The library starts learning from itself.

Based on the brain's neuroplasticity process. In neuroscience, neuroplasticity is how the cortex rewires itself as a side effect of normal thinking — neurons that fire together strengthen their synapse (Hebb, 1949), unused pathways atrophy, and structural rewiring happens offline during sleep (Tononi & Cirelli, 2014). The cortex doesn't run a batch job to keep itself tidy — the rewiring is part of the thinking.

🧠
In the brain
  • A neuron fires — every signal passing through a synapse is a local event.
  • Coincident firing strengthens the synapse — cells that fire together wire together.
  • Unused pathways fade over time.
  • Sleep consolidates: structural rewiring runs offline from perception, on a slower clock.
🌐
In KAOS
  • Every record_outcome(), every memory hit, every agent completion is a "firing".
  • Skills used in the same agent session get a Hebbian edge in the association graph.
  • Cold skills get flagged for pruning; dead fixes auto-downgrade.
  • Structural consolidation (promote / prune / merge) runs in batch at agent-completion — our "sleep".

The claim is an architectural analogy, not a neurobiological one: we borrow the scale separation between fast synaptic updates and slow structural consolidation. The algorithms are classical — Wilson lower bounds, exponential decay, Jaccard similarity, sliding-window counters — arranged to flow continuously through normal agent operation.

What that gives us in practice: the skill library, memory store, and shared log have been collecting real signal for a while — use_count, success_count, every retrieval, every completion. v0.8.1 makes KAOS read that signal continuously and reshape retrieval rankings, build a co-occurrence graph, and triage failures automatically.

Six scenes. Real CLI. Reproducible via uv run kaos skills search --rank weighted + kaos dream failures.

Why Production Agentic Systems Are Still Hard

The research on every individual problem exists. The problem is nobody has put the pieces together into a coherent, composable system.

Research is scattered

Memory isolation from one paper. Coordination protocols from another. Context compression from a third repo. Each solves one problem — none were designed to work together.

Agents step on each other

Two agents write to the same file. One overwrites the other's work. You don't find out until production. There's a proven solution: enforced per-agent VFS.

Agents repeat past mistakes

Every session starts from zero. The agent hits the same JSON parse error it hit last week. Cross-session memory with FTS5 search is a solved problem — just not integrated anywhere.

Agents act without oversight

An autonomous agent drops a production table. There's an established protocol for consensus-based coordination — intent, vote, decide — but nobody ships it.

You can't roll back one agent

The refactor agent broke everything. You git reset --hard and lose the work of the 3 other agents that were fine. Surgical checkpoint/restore is a solved problem.

Context explodes, quality doesn't

Agents bloat their context with raw traces. AAAK compact notation cuts 57% at default with zero quality loss — proven across 5 domains — and it sits unused in a GitHub repo.

You can't debug what happened

47 tool calls, 12 files modified, one broken codebase. An append-only event journal with SQL queries is the right answer. It just needs to be the default.

One Framework. Every Solved Problem.

KAOS doesn’t solve these problems from scratch. It identifies who already solved them — and integrates their solutions into a single, composable system.

 example.py 
from kaos import Kaos

db = Kaos("project.db")

# Each agent is isolated — they cannot see each other's files
agent_a = db.spawn("refactorer")
agent_b = db.spawn("test-writer")

db.write(agent_a, "/src/auth.py", b"# refactored auth module")
db.write(agent_b, "/src/auth.py", b"# test stubs for auth")
# Both wrote to "/src/auth.py" — no conflict. Each has their own copy.

# Checkpoint before risky operations
cp = db.checkpoint(agent_a, label="before-migration")
# ... agent does something dangerous ...
db.restore(agent_a, cp)  # roll back just this agent, others untouched

# What exactly did it do? Query the audit trail with SQL.
db.query("SELECT event_type, payload FROM events WHERE agent_id = ?", [agent_a])

Everything lives in one .db file. Copy it to back up. Send it to a teammate. Query with any SQLite client.

Checkpoint / Restore / Diff

Intelligent Model Routing

Content-Addressable Blobs

Single-File Portability

Parallel Execution

No agent can execute a risky action without consensus. A human supervisor can veto. The rejection reason is saved to memory so the agent learns not to propose the same thing again.
